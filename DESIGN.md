## 2. RBAC 权限模型

### 2.1 模型设计

```
┌─────────────────────────────────────────────────────────────┐
│                        用户 (User)                          │
│  - id, username, email, password, enabled                  │
└────────────────────────┬────────────────────────────────────┘
                        │
                        │ N:M
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                     用户角色 (UserRole)                      │
└────────────────────────┬────────────────────────────────────┘
                        │
                        │ N:M
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                       角色 (Role)                           │
│  - id, name, description                                   │
│  - ADMIN, USER, MANAGER                                    │
└────────────────────────┬────────────────────────────────────┘
                        │
                        │ N:M
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                   角色权限 (RolePermission)                  │
└────────────────────────┬────────────────────────────────────┘
                        │
                        │ N:M
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                     权限 (Permission)                        │
│  - id, name, description                                   │
│  - 格式: resource:action                                   │
│  - user:read, user:write, user:delete, admin:manage        │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 权限说明

| 权限 | 说明 | 角色 |
|------|------|------|
| `user:read` | 读取用户信息 | USER, ADMIN, MANAGER |
| `user:write` | 创建/修改用户 | ADMIN |
| `user:delete` | 删除用户 | ADMIN |
| `admin:manage` | 管理员权限 | ADMIN |
| `order:read` | 读取订单 | MANAGER |
| `order:write` | 创建/修改订单 | MANAGER |
| `product:read` | 读取商品 | MANAGER |
| `product:write` | 管理商品 | MANAGER |

## 3. 认证机制

### 3.1 认证类型

| 类型 | 适用场景 | 凭证 |
|------|---------|------|
| **人机用户** | 前端/移动端用户 | 用户名 + 密码 |
| **机机账户** | 系统间调用 (M2M) | AccessKey + SecretKey |

### 3.2 JWT Token 结构

```json
{
  "sub": "username",
  "userId": 123,
  "roles": ["USER"],
  "permissions": ["user:read"],
  "iat": 1700000000,
  "exp": 1700086400
}
```

### 3.3 机机账户 AKSK

- **AccessKey**: `ak_xxxxxxxxxxxx` - 公开标识
- **SecretKey**: 随机生成，仅创建时返回一次
- **支持轮转**: 验证当前密钥后可生成新密钥

## 4. 安全特性

### 4.1 已实现

- [x] JWT Token 认证
- [x] 密码 BCrypt 加密存储
- [x] 密码强度验证（至少8位数字+字母）
- [x] AKSK 认证（机机账户）
- [x] AKSK 轮转机制
- [x] 敏感信息环境变量配置
- [x] Spring Security 权限控制
- [x] 方法级权限注解 (`@PreAuthorize`)

### 4.2 安全建议

- 生产环境修改默认 JWT Secret
- 定期轮转数据库密码
- 启用 HTTPS
- 配置防火墙规则
- 启用审计日志
