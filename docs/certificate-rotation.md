## 二、轮转策略

### 2.1 证书有效期

| 证书类型 | 有效期 | 轮转提前期 | 轮转方式 |
|---------|-------|-----------|---------|
| Root CA | 20 年 | 6 个月 | 手动更换 |
| Issuing CA | 2 年 | 1 个月 | 自动签发 |
| 服务证书 | 90 天 | 30 天 | 自动轮转 |
| 客户端证书 | 90 天 | 30 天 | 自动轮转 |

### 2.2 轮转流程

```
证书到期前30天
       │
       ▼
┌──────────────┐
│  生成新证书  │ ← 调用证书签发服务
└──────────────┘
       │
       ▼
┌──────────────┐
│  部署新证书  │ ← 双证书并行（新旧同时生效）
└──────────────┘
       │
       ▼
┌──────────────┐
│  验证服务    │ ← 健康检查 + 流量验证
└──────────────┘
       │
       ▼
┌──────────────┐
│  移除旧证书  │ ← 新证书稳定后清理
└──────────────┘
```

### 2.3 轮转时间线

| 天数 | 操作 | 状态 |
|------|------|------|
| Day 0 | 颁发证书 | 正常 |
| Day 60 | 生成新证书 | 正常 |
| Day 65 | 部署到测试环境 | 测试 |
| Day 70 | 部署到生产环境 | 验证 |
| Day 90 | 旧证书过期 | 轮转完成 |

## 三、安全架构

### 3.1 传输层安全

**TLS 版本**
- 强制 TLS 1.3
- 兼容 TLS 1.2（降级方案）

**加密套件**
```
TLS_AES_256_GCM_SHA384 (TLS 1.3)
TLS_AES_128_GCM_SHA256 (TLS 1.3)
TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384 (TLS 1.2)
TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256 (TLS 1.2)
```

**禁用项**
- TLS 1.0 / 1.1
- 所有 RC4、DES、3DES 密码套件
- SSL 协议

### 3.2 双向认证

```
┌─────────────┐     双向认证      ┌─────────────┐
│  Client    │ ◄──────────────► │   Server    │
│             │  1. 客户端证书   │              │
│             │  2. 服务端证书   │              │
│             │  3. 证书链验证    │              │
└─────────────┘                  └─────────────┘
```

### 3.3 证书固定（可选）

## 四、组件设计

### 4.1 证书管理服务

```yaml
CertificateManager:
  功能:
    - 证书签发
    - 证书吊销
    - 证书轮转
    - CRL 管理
    - 证书查询
    
  接口:
    - /api/certs/issue     # 签发证书
    - /api/certs/revoke    # 吊销证书
    - /api/certs/rotate    # 轮转证书
    - /api/certs/crl       # 获取 CRL
    
  存储:
    - MySQL: 证书元数据
    - Vault: 私钥存储
```

### 4.2 证书轮转服务

```yaml
CertificateRotationService:
  触发条件:
    - 定时任务（每小时检查）
    - 证书到期前 30 天
    - 手动触发
    
  执行流程:
    1. 检查证书到期时间
    2. 生成新证书 CSR
    3. 提交给 CA 签发
    4. 部署到目标服务
    5. 验证服务可用性
    6. 记录审计日志
    7. 清理旧证书
```

### 4.3 监控指标

| 指标名 | 类型 | 描述 |
|--------|------|------|
| `cert_expiry_days` | Gauge | 证书到期剩余天数 |
| `cert_rotation_total` | Counter | 证书轮转总次数 |
| `cert_rotation_errors` | Counter | 轮转失败次数 |
| `cert_validation_errors` | Counter | 证书验证错误次数 |

### 5.1 双 CA 架构

```
┌─────────────────┐        ┌─────────────────┐
│   Primary CA    │        │  Secondary CA  │
│  (当前使用)     │        │   (灾备使用)    │
└────────┬────────┘        └────────┬────────┘
         │                          │
         └──────────┬───────────────┘
                    ▼
           ┌─────────────────┐

           │   (双写策略)   │
           └────────┬────────┘
                    │
         ┌──────────┼──────────┐
         ▼          ▼          ▼
    ┌─────────┐┌─────────┐┌─────────┐
    │ Eureka  ││  User   ││ Order  │
    │ Server  ││ Service ││Service │
    └─────────┘└─────────┘└─────────┘
```

### 5.2 故障切换策略

| 场景 | 处理方案 |
|------|---------|
| 主 CA 故障 | 自动切换到备用 CA |
| 证书轮转失败 | 保留旧证书，告警人工介入 |
| 证书链验证失败 | 拒绝连接，回滚到旧证书 |
| CRL 获取失败 | 使用缓存的 CRL（最大 24 小时） |

```

   │
   ▼

   │
   ▼
```

| 场景 | 响应时间 | 操作 |
|------|---------|------|

## 十、附录

### 10.1 常见问题

**Q: 轮转期间服务会中断吗？**
A: 不会。采用双证书并行机制，新证书生效后再移除旧证书。

**Q: 支持哪些密钥算法？**
A: 推荐 RSA 2048/4096 或 ECDSA P-256/P-384。

**Q: 如何处理证书吊销？**
A: 使用 CRL（证书吊销列表）或 OCSP（在线证书状态协议）。
